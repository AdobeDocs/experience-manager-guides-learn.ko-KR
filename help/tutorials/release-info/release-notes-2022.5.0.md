# Adobe Experience Manager 안내서 5월 릴리스 as a Cloud Service

## 5월 릴리스로 업그레이드

현재 Adobe Experience Manager 가이드를 as a Cloud Service(라고 함)로 업그레이드합니다 *AEM 안내서 as a Cloud Service*) 다음 단계를 수행하여 설정합니다.
1. Cloud Services의 Git 코드를 확인하고 업그레이드하려는 환경에 해당하는 Cloud Services 파이프라인에 구성된 분기로 전환합니다.
1. 업데이트 `<dox.version>` 속성 `/dox/dox.installer/pom.xml` Cloud Services Git 코드 파일을 2022.5.144에 추가합니다.
1. 변경 사항을 커밋하고 Cloud Services 파이프라인을 실행하여 AEM Guides as a Cloud Service 5월 릴리스로 업그레이드합니다.

## 새로운 기능 및 향상된 기능

AEM 안내서는 5월 릴리스의 다양한 개선 사항과 새로운 기능을 제공합니다.

### 향상된 웹 편집기

* **사용자 지정된 템플릿을 기반으로 맵 만들기**

이제 사용자 지정된 맵 템플릿을 만드는 강력한 기능이 제공됩니다. 이를 사용하여 주제 템플릿과 함께 DITA 맵을 만들고 맵 템플릿에서 참조하는 템플릿 맵을 매핑할 수 있습니다.

![dita 템플릿](assets/dita-templates.png)

사용자 지정된 맵 템플릿에서 다른 맵 템플릿 및 주제 템플릿을 참조할 수도 있습니다. 참조되는 맵 템플릿은 다양한 맵 템플릿, 항목 템플릿, 주제, 맵, 이미지, 비디오 및 기타 자산을 의미할 수 있습니다.

![dita 템플릿 만들기](assets/create-dita-template.png)

사용자 지정된 맵 템플릿을 사용하면 맵 템플릿과 전체 참조 폴더 구조를 매우 쉽게 복제할 수 있습니다. 이러한 사용자 지정된 템플릿은 재귀 구조 및 참조가 있는 여러 맵을 만들고 다시 만드는 데 특히 유용합니다.

* 다음 **키워드 삽입** 기능이 개선되었습니다. 이제 키워드가 알파벳 순서로 나열되므로 삽입할 키워드를 보다 쉽게 찾을 수 있습니다. 검색 상자에 검색 문자열을 입력하여 키워드를 검색할 수도 있습니다.

![키워드 삽입](assets/insert-keyword.png)

* 이제 저장소 보기 파일에서 파일은 배치로 로드됩니다. 한 번에 75개의 파일이 로드됩니다. 이 배치 로드는 효율적이며 폴더에 있는 모든 파일을 로드하는 것과 비교하여 파일에 더 빨리 액세스할 수 있습니다.

![더 많은 파일 로드](assets/load-more-files.png)

* 미리 보기 및 작성자 보기를 포함하되 이에 제한되지 않는 모든 XML 편집기 화면에서 포함된 데이터 또는 링크를 포함하는 SVG 이미지를 렌더링할 수 있습니다.

* 기본 XSD/DTD를 최신 버전으로 업데이트할 수 있습니다

### 향상된 번역 프로세스

* **범위 번역 프로젝트를 만드는 기능**
변환할 프로젝트의 범위만 만들어야 하는 경우 선택할 수 있습니다 
**새 범위 번역 프로젝트 만들기**. 이렇게 하면 번역용 복사본이 전송되지 않고 파일의 원래 번역 상태가 유지됩니다.

![번역 프로젝트 반복](assets/scoping-translation-project.png)

* 번역 작업에서 하나 이상의 주제에 대한 번역을 거부하면 거부된 모든 주제에 대한 진행 중 번역 상태는 원래 상태로 되돌아갑니다.

* 다음 **언어** 목록에 언어 폴더와 해당 언어 코드가 표시됩니다. 예를 들어, 프랑스어(fr) 및 독일어(de)입니다.

* 이제 번역 기능은 국가와 언어를 모두 포함하는 언어 코드도 지원합니다. 예, `fr-fr`, `en-us`.

![번역어](assets/translation-languages.png)

* 언어 폴더 외부에 있는 DITA 맵을 로드할 때 백엔드에 예외가 기록되지 않습니다.

번역에 대한 자세한 내용은 *웹 편집기에서 문서 번역* 섹션을 참조하십시오.


### 향상된 게시

* 또한 **대시보드 게시** 맵 대시보드에서 출력을 생성하는 동안 출력 탭에서 출력을 클릭합니다. 모든 활성 게시 작업 목록은 게시 대시보드에서 사용할 수 있습니다.

![큐 출력](assets/queued-output.png)

* 맵 대시보드에서 여러 DITAVAL 파일을 선택하여 조건부 컨텐츠를 생성할 수 있습니다. 파일을 추가하거나 삭제하여 파일 순서를 유지할 수 있습니다. 파일 이름 위로 마우스를 가져가면 파일이 저장된 AEM 저장소의 경로를 볼 수 있습니다.

* **사용되지 않는 기능**
AEM as a Cloud Service은 더 이상 FrameMaker 문서에 대한 DITA 출력 형식 생성을 지원하지 않습니다. 이 DITA 옵션은 맵 대시보드의 출력 사전 설정에서 제거되었습니다.

### 향상된 문서 기반 게시

XML Editor에서는 Salesforce 프로필에 게시하는 동안 두 개 이상의 제품 범주를 문서에 매핑하는 기능을 제공합니다.

### 기타 기능 개선 사항

* 미리 보기 모드도 지원합니다 `deliveryTarget` DITA의 조건부 처리 특성. 이 옵션은 다음과 함께 드롭다운 필터에서 옵션으로 사용할 수 있습니다 **대상자**, **platform**, **product**, props, **otherprop**.
* Oxygen의 AEM 서버와 로컬 시스템 간에 강제로 동기화하는 옵션이 제공되었습니다.

## 해결된 문제

다음과 같이 다양한 영역에서 해결된 버그가 있습니다.

* 웹 편집기의 검토 패널에서는 사용자가 검토 주석에 회신할 수 없습니다. (9667)
* 옵션 메뉴를 통해 빈 폴더를 새로 고친 후 빈 폴더를 클릭하면 응용 프로그램이 비어 있게 됩니다. (9639)
* 새 버전은 **저장 후 닫기** 체크 인 파일입니다. (9638)
* 닫기 버튼은 **새 버전으로 저장** 확인란을 사용할 수 있습니다. (9637)
* 올바른 PDF이 각 장에 대한 별도의 PDF을 통해 처음 게시되고 단일 PDF 파일을 통해 게시되는 경우 게시되지 않습니다(별도의 PDF 파일 만들기가 선택 취소됨). (9632)
* 맵 대시보드에서 관리자가 아닌 사용자에게 메타데이터 문제를 발생합니다. (9620)
* 기준선을 만들면 서버에 파일이 10000개 이상 있는 경우 UI에서 상태가 실패(상태 가져오기 호출이 실패)로 설정됩니다. (9608)
* 분할 게시 워크플로우가 실패할 때 속성에 큰 데이터를 저장하면 게시 오류가 발생합니다. (9586)
* 조건부 속성 필터 상태는 미리 보기 와 다시 미리 보기 모드 간을 전환할 때 유지되지 않습니다. (9553)
* 을 통해 이름을 지정하지 않는 경우 저장소 보기에서 북맵 이름이 비어 있게 됩니다. `mainbooktitle` 태그에 가깝게 포함했습니다. (9538)
* Oxygen을 사용하여 업로드된 파일을 여는 동안 HTTP 400 오류가 발생합니다. (9535)
* 이전에 연 맵의 사전 설정은 사전 설정이 정의되지 않은 맵을 열 때 출력 탭에 계속 표시됩니다. (9523)
* [개요] 패널에서 태그 및 속성에 대한 검색 기능이 작동하지 않습니다. (9506)
* 브라우저를 새로 고칠 때까지 새로 만든 컬렉션이 표시되지 않습니다. (9505)
* 조건부 속성 레이블(값이 아님)이 &quot;모든 prop 추가&quot; 옵션을 통해 모든 조건을 추가할 때 소스 모드에 표시됩니다. (9501)
* 어떤 버전으로 되돌릴 때 파일이 자동으로 체크 아웃됩니다. (9482)
* 파일 버전 복구 시 Assets UI에 잘못된 타임스탬프 차이가 표시됩니다. (9480)
* DITA 맵의 topicref 요소에 항목을 삽입하는 동안 검색 결과의 여러 항목이 추가됩니다. (9474)
* 설정이 **업로드된 파일에 대한 새 버전 만들기** 이(가) 켜진 경우, 고정된 노드에서 복원 및 저장 시 새 버전이 만들어집니다. (9473)
* 키 참조를 정의하는 동안 표시 텍스트가 추가되지 않는 경우 링크 URL 변경 시 키 참조 및 컨텐츠 키 참조의 표시 텍스트가 유지되지 않습니다. (9458)
* 버전 내역에서 버전 번호와 레이블이 현재 버전에 대해 표시되지 않습니다. (9446)
* 편집기에서 특정 컨텐츠 파일을 열면 편집기가 중지됩니다. (9443)
* 저장소 패널에서 검색 및 상위 찾아보기 대화 상자는 컨텐츠가 클 때 화면을 중지합니다. (9432)
* AEM 사이트 출력에 전달된 메타데이터는 컨텐츠의 기준선을 준수하지 않습니다. (9416)
* AEM에서 버전이 되돌린 후 Oxygen이 잘못된 버전의 항목을 확인합니다. (9411)
* 실패한 베이스라인은 맵 대시보드의 사전 설정 탭에서 편집을 비활성화합니다. (9403)
* 새 콘텐츠를 만들 때 항상 오류가 기록됩니다. (9388)
* 새로 만든 DITA 자산은 항상 다른 사용자가 체크 아웃합니다. (9387)
* topicref를 glossref로 변환하는 동안 요소 이름 바꾸기 기능이 제대로 작동하지 않습니다. (9380)
* 버전 레이블이 드롭다운으로 표시되지 않습니다. **새 버전으로 저장** 대화 상자. (9379)
* 다른 버전 간의 전환 시 조건이 적용되지 않습니다. **비교 표시** 드롭다운. (9366)
* 미리 보기 필터를 사용하면 여러 문제가 발생합니다. (9365)
* topicref에 비DITA 및 DITAVAL 자산을 삽입할 수 없습니다. (9363)
* 승인된 번역은 대상 언어 코드에 다음과 같은 5개의 문자가 포함된 경우 대상 언어와 통합되지 않습니다 `fr_ca`. (9357)
* 다음을 사용하여 파일을 검색할 수 없음 **폴더에서 파일 찾기** 에서 **추가 옵션** 메뉴 및 앱이 응답하지 않습니다. (9337)
* 키가 많으면 찾아보기 대화 상자가 중단됩니다. (9332)
* 문서 기반 게시를 수행하는 동안 DITAVAL 파일이 작동하지 않습니다. (9330)
* AEM Site 출력에서 각주 순서가 잘못되었습니다. (9327)
* 선택 경로가 변경될 때 검색이 자동으로 수행되지 않습니다. (9323)
* 번역이 완료되면 번역된 자산에 대해 추가 버전이 만들어집니다. (9310)
* 폴더 프로필에서 관리자 사용자를 삭제할 수 없습니다. (9306)
* 컨텐츠 키 참조에서 업데이트된 루트 맵은 페이지를 새로 고칠 때까지 설정되지 않습니다. (9302)
* Oxygen에서 웹 인증이 작동하지 않습니다. (9296)
* 인코딩된 문자가 있는 웹 링크가 제대로 작동하지 않습니다. (9227)
* Oxygen에서 파일을 열면 체크 아웃되지 않습니다. (9217)

* 번역 및 기준선 탭은 맵 대시보드에서 한동안 표시됩니다. (9146)
* 폴더 프로필을 다시 로드할 때 경험 또는 기능 문제가 발생합니다. (9103)
* 페이지 레이아웃 편집기를 삭제해도 작성자 보기의 가운데 패널에서 닫히지 않습니다. (9087)
* 이미지를 제거한 다음 문서의 새 버전을 저장할 때 웹 편집기에서 유효성 검사 오류가 발생합니다. (8985)
* 다음을 모두 볼 수 없습니다. `glossrefs` 용어집 패널(컨텐츠별)에서 생성합니다. (8886)
* `xref` 텍스트가 없는 경우 문서 기반 게시 출력에 표시되지 않습니다. (8764)
* 참조가 파일 이름에 공백이 있는 움직이는 이미지 또는 멀티미디어 파일에서 중단됩니다. (8624)
* 선택 시 참조 중단이 발생합니다 `Select All` 멀티미디어 파일 또는 DITA 콘텐츠를 다른 폴더로 이동 (8622)
* &#39;대기 중&#39; 또는 &#39;실행 중&#39;과 같은 상태가 있는 출력 작업은 게시 대시보드에서 정리되지 않습니다.  (8569)

* 각 빈 공간에 예기치 않은 공간이 있습니다. `entry` outputclass 속성이 다음에 추가될 때 요소 `tgroup` 요소를 생성하지 않습니다. (7532)
* 작업이 완료되는 즉시 저장소 패널에 체크 인되거나 체크 아웃된 파일 잠금 아이콘이 표시되지 않습니다. (5817)
* 편집기에서 파일을 체크 인하더라도 잠금 아이콘이 저장소 보기에 표시됩니다.  (5756)
* [출력] 탭 아래에 AEM 사전 설정에서 사이트가 누락되었습니다. (9567)
* 일부 DITA 파일을 편집하려고 하면 XML 편집기가 중단됩니다. (9537)

* 파일이 동일한 위치에 AEM에 이미 있으면 Oxygen에서 다시 업로드가 실패합니다. (9328)

* Oxygen에서 AEM으로 문서를 체크 인하면 문서의 일본어 컨텐츠가 물음표(???)으로 바뀝니다. (9276)
