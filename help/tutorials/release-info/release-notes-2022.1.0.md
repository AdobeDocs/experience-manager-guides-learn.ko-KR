## 새로운 기능 및 향상된 기능

### 문서 기반 게시

1월 릴리스에서는 웹 편집기 내에 통합된 문서 기반 게시 기능을 도입했습니다. 문서 기반 게시 기능을 사용하여 하나 이상의 주제에 대한 출력을 점진적으로 생성하거나 컨텐츠를 기술 자료 플랫폼에 게시할 수 있습니다.

이 기능을 사용하면 추가 방식으로 DITA 맵을 빌드하고 주제를 준비했을 때 게시할 수 있습니다. 맵을 게시하면 문서 기반 게시 기능을 사용하여 업데이트된 문서에 대해서만 증분 게시를 수행합니다.

![문서 기반 게시](assets/article-based-publishing.png)

AEM 외에도 이 고유한 기능을 사용하여 Salesforce와 같은 기술 자료 포털에 문서를 게시할 수 있습니다. 이 기능은 AEM 핵심 구성 요소 위에 구축된 OOTB 컨텐츠 템플릿과 함께 제공됩니다. 이 템플릿을 사용하면 기술 컨텐츠에 대한 지식 기반 저장소를 만들 수 있습니다. 이 템플릿의 장점은 조직 요구 사항에 맞게 완벽하게 사용자 지정할 수 있으며 기업 인트라넷 포털과 같은 사용 사례를 지원할 수 있다는 것입니다.
문서 상태 및 수정된 시간에 따라 문서를 필터링할 수도 있습니다.

이 이동 중에 필요한 문서 게시 기능을 사용하면 컨텐츠 게시에 대한 전체 제어 권한을 제공할 뿐만 아니라 업데이트된 컨텐츠를 게시하는 전체 시간을 줄일 수 있습니다.
이 템플릿을 사용하여 문서를 게시하면 게시된 페이지에 메타데이터를 전달할 수도 있습니다.
자세한 내용은 *웹 편집기에서 문서 기반 게시* 을 참조하십시오.

### 향상된 웹 편집기

웹 편집기에 도입된 다양한 개선 사항과 새로운 기능이 있습니다.

* 제목 구성표에 대한 지원도 웹 편집기에 추가되었습니다. 이제 제목 체계 패널을 사용하여 제목 체계를 만들고 사용할 수 있습니다. 주제 체계를 추가하면 이제 고유한 회사 메타데이터 및 분류법을 사용할 수 있습니다.

![주제 체계](assets/subject-scheme-panel.png)

* 용어집을 일괄적으로 관리하기 위해 이 버전에 새로운 핫스팟 도구가 도입되었습니다. 이 도구를 사용하면 선택한 맵이나 열려 있는 주제에 대해 텍스트를 용어집 및 용어집으로 빠르게 변환할 수 있습니다.

![용어집 핫스팟](assets/glossary-hotspot-tool.png)

* 참조 파일에서 재사용 가능한 컨텐츠를 빠르게 새로 고칠 수 있는 새로 고침 기능을 재사용 가능한 콘텐츠 패널에 추가했습니다.
* 새 작업 복사 표시기에는 파일의 현재(작업 복사본)이 저장된 버전과 동기화되어 있는지 여부가 표시됩니다.

![버전 표시기](assets/version-update-indicator.png)

* 저장소 패널 및 파일 찾아보기 대화 상자의 검색 필터가 개선되어 추가 사용자 지정할 수 있는 추가 필터링 옵션을 제공합니다.

![저장소에서 검색 필터](assets/repository-filter-search.png)

* 이제 웹 편집기에서 .docx 파일을 업로드할 수 있습니다.

### FrameMaker를 사용한 작성자

이제 FrameMaker에서 문서를 작성하고 게시할 수 있습니다. FrameMaker는 Adobe Experience Manager에 기본 제공 커넥터와 함께 제공됩니다. FrameMaker에는 분산되어 공동 작업을 수행하는 환경에서 문서의 버전을 유지할 수 있는 사용하기 쉬운 인터페이스가 제공됩니다.

컨텐츠를 만들면 FrameMaker에서 PDF, HTML5, EPUB 및 DITA와 같은 다양한 형식으로 문서를 게시할 수 있습니다. 체크 아웃, 종속 항목이 있는 체크 아웃, 체크인, 새로 고침 등과 같은 다양한 파일 관리 작업을 수행할 수도 있습니다.
FrameMaker를 사용하여 [!DNL AEM Guides] as a Cloud Service에서 FrameMaker 버전 2020.4 이상을 사용합니다.

### 새 번역 대시보드

다음 기능과 함께 웹 편집기에 새로운 번역 대시보드가 도입되었습니다.

* 주제 목록 정렬, 검색 및 필터링.
* 참조 유형별 컨텐츠 필터링 - 직접 또는 간접 참조
* 번역 요청을 시작할 때 기존 프로젝트를 쉽게 찾을 수 있습니다.
* 두 개 이상의 언어에 대한 번역 요청이 시작되면 각 언어에 대해 여러 프로젝트를 만들 수 없도록 하는 다국어 번역 메커니즘이 도입되었습니다.
* 맵 대시보드에서 번역 탭을 숨기기 위한 구성이 도입되었습니다. 기본적으로 표시됩니다. 맵 대시보드 또는 웹 편집기를 사용하여 컨텐츠를 번역하도록 선택할 수 있습니다.

![번역 대시보드](assets/translation-from-web-editor.png)

### 향상된 게시

* 이제 작성자가 DITA-OT 게시에 맵 및 주제 수준 메타데이터를 전달할 수 있습니다. 이 기능은 사용자 지정 PDF 템플릿이 태그, 작성자, 문서 상태 등과 같은 파일 메타데이터 속성을 사용하도록 디자인될 때 유용합니다.

![DITA-OT 메타데이터](assets/custom-meta-data-output-preset.png)

* 사용자가 삭제될 때 해당 버전의 항목을 유지하거나 삭제할 수 있도록 하는 새 구성이 추가되었습니다 **삭제 및 만들기** 옵션은 AEM 사이트 출력 생성에서 사용됩니다.

### 향상된 파일 처리

이제 AEM Assets에서 파일을 사용하여 작업하는 동안 다음 개선 사항을 볼 수 있습니다.
* 새로운 파일 업로드 경험 및 충돌 해결 전략을 선택하기 위한 새로운 대화 상자가 도입되었습니다.

![파일 업로드 충돌](assets/file-upload-name-conflict.png)

* 체크 아웃된 파일을 덮어쓰지 않도록 하는 기능을 사용하여 업로드된 파일의 새 버전을 만들 수 있습니다.
* 이제 버전 기록 보기에서 직접 이미지 미리 보기를 볼 수 있습니다. 또한 DITA 및 비DITA 파일의 경우 버전 기록에 현재 버전 정보가 별도로 표시됩니다.

![버전 기록 축소판](assets/version-history-preview-image.png)

* 사용자가 DITA 파일을 작성할 때마다 기본 파일 이름이 작은 대시에 표시되어 Native AEM 폴더 작성 시나리오로 인라인으로 만듭니다.

### 새로운 보고서 내보내기 기능

보고서는 컨텐츠의 상태를 식별하는 데 매우 유용합니다. [!DNL AEM Guides] as a Cloud Service에서는 컨텐츠를 제어할 수 있는 다양한 보고서를 제공합니다. 이제, 보고서를 볼 수 있을 뿐만 아니라, 보고서 데이터를 CSV 파일로 내보내 더 큰 팀과 보고 공유할 수도 있습니다. 보고서 데이터를 통해 끊어진 링크 또는 누락된 이미지를 한눈에 볼 수 있습니다.

![보고서 내보내기](assets/export-report.png)

### 향상된 산소 DAM 새로 고침 환경

Oxygen에서 AEM Server의 파일을 새로 고치면 현재 Oxygen 세션에 저장되지 않은 파일이 있는 경우 경고 메시지가 표시됩니다. 저장하지 않은 파일을 저장하려면 새로 고침 작업을 취소하도록 선택할 수 있습니다. 이 기능이 없으면 사용자가 문서에서 저장하지 않은 정보를 손실했습니다.


### 기타 기능 개선 사항

* 이제 새 항목을 만들 수 있습니다 **Dita 프로젝트** 아래의 템플릿 **/apps/projects/templates** 경로.
* 이제 기본값을 다운로드합니다 **ui_config.json** 파일을 폴더 프로필에서 가져올 수 있습니다. 기존 사용자 지정 변경 사항을 병합하는 데 사용할 수 있습니다 **ui_config.json** 파일을 업그레이드하는 동안 파일을 생성합니다.
* 새 버전의 JS 파일이 있는 경우에도 브라우저 캐시를 지울 필요가 없습니다.

## 해결된 문제

다음과 같이 다양한 영역에서 해결된 버그가 있습니다.

### 웹 편집기

* Conrefs는 손상되지 않은 경우에도 빨간색 색상으로 표시됩니다. (8239)
* DITAVAL 편집기에서 모든 속성 추가 를 선택하면 조건부 속성의 값이 자동으로 채워지지 않습니다. (8234)
* 작성자는 상대 경로를 사용하여 주제에 이미지를 삽입할 수 없습니다. (8112)
* 파일 이름에 공백이 있는 경우 작업 페이지에 멀티미디어 파일이 표시되지 않습니다. (8111)
* 테이블 셀에 추가된 Ph상선은 빨간색 색상으로 표시됩니다. (8083)
* 검토 중인 파일을 이동해도 검토 작업의 링크가 업데이트되지 않습니다. (8080)
* 크기 조절 속성이 75% 이상으로 설정된 이미지를 웹 편집기에서 올바르게 렌더링하지 않습니다. (8073)
* GIF 이미지는 웹 편집기에서 정적 이미지로 렌더링됩니다. (8024)
* 참고 요소의 conkeyref는 웹 편집기 미리 보기나 출력에 표시되지 않습니다. (8006)
* conref인 요소에 대한 xref는 편집기에서 해결되지 않습니다. (7933)
* 키가 있는 제목이 편집기 미리 보기 및 저장소 패널에서 올바르게 렌더링되지 않습니다. (7909)
* 특수 문자가 있는 코드 조각이 올바르게 저장되지 않습니다. (7908)
* MathML 수식의 서식을 지정한 후 항목을 저장하면 오류가 발생합니다. (7954)
* tm이 있는 keydef가 편집기에서 제대로 렌더링되지 않으며 AEM 사이트 출력에 중복 TM 기호가 포함되어 있습니다. (7859)
* 코드 조각을 드래그하여 놓는 것은 DTD에 따라 작동하지 않습니다. (7758)
* HTML에서 그래픽에 대해 사용자 정의된 차원을 무시합니다. (7718)
* 원본 파일을 이동해도 conrefend 특성이 업데이트되지 않습니다. (7698)
* 참조 항목 유형 문서를 사용하면 여러 UI 문제가 발생합니다. (7656)
* 작성자가 맵에서 ditavalref를 추가하면 DITAVAL 파일이 표시되지 않습니다. (7594)
* 공백이 각각 있습니다 `<entry>` outputclass 속성이 다음에 추가될 때 요소 `<tgroup>` 요소를 생성하지 않습니다. (7532)
* 맵 대시보드를 통해 열린 항목에 대해서는 소스 단추가 작동하지 않습니다. (7465)
* 예쁜 인쇄에서는 FrameMaker 또는 Oxygen에서 파일을 열 때 볼 수 있는 빈 선과 공백을 삽입합니다. (7408)
* 주제 중 어느 것이든 href=&quot;/&quot;가 있는 맵은 AEM 사이트에 게시하지 않습니다. (7405)
* 루트 맵에 많은 수의 키가 있는 경우 편집기에서 발견된 성능 문제입니다. (7400)
* 사용자 지정 템플릿이 있는 맵의 문서 상태가 해당 상태 프로필에서 상속되지 않습니다. (7359)
* `<tm>` 요소가 블록 요소로 잘못 렌더링됩니다. (7286)
* 새 템플릿을 만들면 편집기 템플릿 패널에 중복 템플릿이 표시됩니다. (5814)
* 추가 속성을 설정하기 위해 ui_config에 정의된 템플릿은 드래그/드롭 사례에 적용할 수 없습니다. (5713)
* menuacade에서 uicontrol의 기본 모양이 잘못되었습니다. (5483)
* 주제/맵에 대한 사용자 지정 템플릿에 UI에 새 이름이 표시되지 않습니다. 구성된 이름을 표시하지 않고 &quot;Topic&quot;/&quot;Map&quot;으로 이름을 표시합니다. (4958)
* 사용자 환경 설정 설정에서 로드맵 지우기 기능. (8534)
* 페이지를 새로 고친 후에도 새로 만든 맵 컬렉션이 나열되지 않습니다.(8603)
* 잠금 해제된 항목은 닫을 수 없습니다. (8545)
* 소스와 작성자 모드 간을 전환하면 주제가 더티(dirty)로 표시되고 컨텐츠를 다시 저장해야 합니다.(8524)
* 특수 문자 검색 시 콘텐츠 패널 충돌 재사용 `[` 또는 `*` .(8279)
* 키보드 단축키 Alt+Enter를 사용하여 요소 삽입 대화 상자를 열면 커서가 검색 표시줄에 표시되지 않습니다.(7912)
* 검색 옵션은 파일 이름에서만 검색하며 컨텐츠에서는 검색되지 않습니다. (7784)


### 산소 커넥터

* 상위 폴더에 특수 문자가 있는 파일은 Oxygen에서 로드하는 동안 오류가 발생합니다. (8054)
* 새로 만든 문서가 Oxygen에서 열리면 &quot;GUID를 찾을 수 없음&quot; 오류가 발생합니다. (7856)

### 해결 방법에 대한 알려진 문제

다음과 같은 알려진 문제에 대해 주어진 해결 방법을 사용하십시오.

* Mac의 Oxygen 커넥터에 대해 웹 인증이 작동하지 않습니다.
   **해결 방법**: Windows에서 Oxygen 커넥터를 사용하십시오.

* Firefox 브라우저에서 나란히 보기를 열지 않으면 검토 주석을 가져올 수 없습니다.
   **해결 방법**: 지금은 Chrome 브라우저를 사용합니다.

* 파일 이름에 공백이 있는 이미지 또는 멀티미디어 파일을 이동할 때 참조가 중단됩니다.
   **해결 방법**: 파일 이름을 변경하고 파일 이름에서 공백을 제거한 후 이동합니다.

* 맵 대시보드는 최신 버전의 Chrome 브라우저에서 간헐적으로 로드되지 않습니다.
   **해결 방법**: 맵 대시보드 페이지를 새로 고칩니다.
